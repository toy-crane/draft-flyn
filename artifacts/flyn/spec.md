# Flyn — Spec

> AI와 카톡처럼 영어로 채팅하면서, 내가 보낸 메시지가 맞는지 틀린지 즉시 확인할 수 있는 앱
>
> 작성일: 2026-05-21
> 기반: `artifacts/flyn/decisions.md`

---

## 개요

A2-B1 수준의 한국 학습자가 시나리오 기반 영어 대화를 카카오톡 같은 채팅 UI에서 진행한다. 사용자가 보낸 메시지 옆에는 ✓/✱/💡 마크가 즉시 표시되어 흐름을 끊지 않고 교정이 이뤄지고, 영어가 막히면 한국어로 입력해도 영어로 변환되어 전송된다. 시나리오마다 3개의 학습 목표가 자동으로 부여되며 매 메시지마다 달성 여부가 실시간으로 표시된다.

## 범위

### 포함

- Google OAuth 로그인 (강제, 게스트 없음)
- 좌측 사이드바 + 메인 영역의 레이아웃 (반응형 — 모바일은 햄버거 메뉴로 변환)
- 시나리오 생성/편집/삭제 (자유 텍스트 폼 + 추천 칩 + ✨ 필드별·전체 AI 자동 채움)
- AI 자동 생성 학습 목표 3개 (재생성만 가능, 직접 편집 불가)
- 시나리오 진입 → AI가 먼저 발화 → 사용자와 채팅 진행
- 사용자 메시지 옆 마크 — ✓ OK, ✱ 교정 필요, 💡 대안, 🌐 번역 (메시지당 하나)
- 교정·번역의 라벨드 섹션 (기본 펼침, 접기 가능, 메시지별 상태 유지)
- 한국어 입력 자동 감지 → 전송 전 영어 미리보기 카드 → 영어로 전송
- 채팅 헤더 아래 목표 3개 상시 표시 + 매 메시지마다 달성 평가 + 3개 달성 시 종료 권유
- "더 알아보기" 사이드 패널 (웹: 우측 드로어, 모바일: 바텀시트) — 일회용, 한국어 코치 페르소나
- 진행 중 vs 종료된 시나리오의 사이드바 구분 (진행 중은 ▸ 표시)
- 같은 시나리오 재플레이 (시나리오당 진행 중 인스턴스 최대 1개)
- 종료된 시나리오의 읽기 전용 히스토리 보기
- Chat input: 자동 확장 (최대 5줄), Enter 전송, Shift+Enter 줄바꿈, 한글 IME 안전 처리, 500자 제한, AI 응답 중 비활성화
- 핵심 실패 경로: 채팅 호출 실패 시 "다시 시도" / 분석 호출 실패 시 마크 없이 진행

### 제외

- 익명/게스트 사용 — LLM 비용 통제 불가
- Kakao/Apple 등 추가 OAuth 제공자 — V2
- 결제·freemium·rate limiting — 본인 테스트 빌드라 매출 모델 없음
- 음성 입력 — 도입 시 전체 모델 재설계 필요, 다른 제품으로 분류
- AI 응답에 내레이션/메타 텍스트 ("[smiles]" 등) — 학습 가치 없음
- 학습 목표의 직접 텍스트 편집 — 모호한 수정 시 달성 평가 불가
- 한 메시지에 ✱와 💡 동시 표기 — 메시지당 마크는 하나
- AI 응답 중 다음 메시지 큐잉 — 모델 복잡도 ↑
- 사이드 대화 영속화 — 닫으면 소실 (같은 ✱ 다시 열어 같은 질문하는 행동 드묾)
- 디테일한 온보딩 / 첫 사용자 4가지 기능 소개 — 큰 CTA + 예시 칩이면 충분하다고 가정

## 시나리오

### 1. 첫 진입과 로그인

- **Given** — 처음 방문한 사용자가 로그인되어 있지 않다
- **When** — 앱 URL에 접속한다
- **Then** — 로고, 한 줄 카피, "Continue with Google" 버튼만 있는 화면이 나타나고, 다른 어떤 영역(사이드바·채팅·시나리오 목록)도 보이지 않는다

성공 기준:
- [ ] 비로그인 상태로 어떤 URL 접근 → 로그인 화면으로 이동
- [ ] 로그인 화면에 "Continue with Google" 외의 인증 버튼이 보이지 않는다
- [ ] Google OAuth 성공 → 사이드바 + 메인 영역의 메인 화면으로 이동
- [ ] OAuth 실패 또는 사용자 취소 → 로그인 화면에 머문다

### 2. 첫 사용자의 빈 상태

- **Given** — 방금 로그인했고 생성한 시나리오가 0개다
- **When** — 메인 화면을 본다
- **Then** — 사이드바에는 "+ 새 시나리오" 버튼만 있고, 메인 영역에는 큰 CTA "첫 시나리오를 만들어보세요"와 예시 칩 3-5개가 표시된다

성공 기준:
- [ ] 시나리오 0개 상태에서 사이드바의 "내 시나리오" 섹션과 "히스토리" 섹션은 보이지 않거나 비어 있다
- [ ] 메인 영역에 "첫 시나리오를 만들어보세요" CTA가 보인다
- [ ] CTA 아래에 예시 칩 3-5개 ("카페 주문", "면접 연습", "공항 환승" 등)가 보인다
- [ ] CTA 또는 "+ 새 시나리오" 버튼 클릭 → 시나리오 생성 폼으로 이동

### 3. 시나리오 생성 (수동 입력)

- **Given** — 사용자가 시나리오 생성 폼을 열었다
- **When** — 상황 "카페에서", 상대방 역할 "바리스타", 내 역할 "손님", 메모 "비건이라 우유 못 마심"을 입력하고 "다음" 버튼을 누른다
- **Then** — 로딩 표시 후 미리보기 화면이 나타나고, 1줄 요약("Bluebird Coffee에서 손님으로 라떼 주문하기" 류)과 AI가 생성한 학습 목표 3개 카드(영어 행동 + 한국어 부연)가 표시된다

성공 기준:
- [ ] 상황·상대방 역할·내 역할 중 하나라도 비어 있으면 "다음" 버튼은 비활성화
- [ ] 폼 제출 → 로딩 상태 표시 → 미리보기 화면 진입
- [ ] 미리보기에 학습 목표 카드 3개가 표시되고, 각 카드는 짧은 영어 행동 ("Order a latte") + 한국어 부연 ("음료 주문하기") 형식이다
- [ ] 미리보기에 시나리오 1줄 요약이 표시된다
- [ ] 각 목표 카드에 ↻ (재생성) 버튼이 있고, 목표 카드에 직접 텍스트 편집 가능한 입력 필드는 없다
- [ ] 미리보기에 "시작하기" 버튼이 있다

### 4. 시나리오 생성 (✨ AI 자동 채움 + 맥락 기반 추천 칩)

- **Given** — 시나리오 생성 폼에서 사용자가 상황 필드에 "카페에서"만 입력했다
- **When** — "상대방 역할" 필드 옆의 ✨ 버튼을 누른다
- **Then** — 잠시 후 상대방 역할 필드에 "카페에서" 컨텍스트를 반영한 텍스트("바리스타" 등)가 자동으로 채워진다

성공 기준:
- [ ] 다른 필드가 비어 있어도 ✨ 버튼은 활성화
- [ ] ✨ 버튼 클릭 → 해당 필드만 채워지고 다른 필드는 그대로
- [ ] 폼 헤더의 "AI에게 전부 맡기기" 버튼 클릭 → 모든 필수 필드가 채워진다
- [ ] 추천 칩은 윗 필드들의 값을 컨텍스트로 반영해 생성된다 — 예: 상황 = "카페에서"일 때 "상대방 역할" 칩은 "바리스타", "매니저", "단골 손님" 등 카페 맥락의 역할만 보이고, "내 역할" 칩은 "처음 가는 손님", "단골", "외국인 여행객" 등 카페+바리스타 맥락이 반영된다
- [ ] 윗 필드(들) 중 바로 앞 필드가 비어 있는 동안에는 해당 아래 필드의 추천 칩 영역이 표시되지 않는다 (사용자는 ✨ 버튼으로 채울 수 있음)
- [ ] 첫 필드(상황)는 윗 필드가 없으므로 추천 칩이 표시되지 않는다
- [ ] 윗 필드 값이 바뀌면 아래 필드들의 칩이 갱신된다 — 사용자가 채운 칩 영역은 가로 스크롤로 5-7개 칩이 보인다
- [ ] 추천 칩 클릭 → 해당 텍스트가 필드에 입력된다
- [ ] 메모 필드에는 추천 칩이 없다 (자유 형식 메모)
- [ ] 칩에 "AI 생성"·"맞춤 추천" 같은 별도 라벨은 붙지 않는다 (칩 내용으로만 맥락 반영을 드러낸다)

### 5. 시나리오 생성 취소

- **Given** — 사용자가 시나리오 생성 폼에 일부 입력했다
- **When** — 사이드바 다른 항목을 클릭하거나 뒤로 가기를 시도한다
- **Then** — "이 시나리오를 저장하지 않을까요?" 확인 모달이 나타나고, 취소를 누르면 폼이 남는다

성공 기준:
- [ ] 폼에 일부라도 입력한 상태에서 이탈 시도 → 확인 모달
- [ ] 모달에서 "저장 안 함" → 입력 내용 폐기 후 이동
- [ ] 모달에서 "계속 작성" → 폼 그대로 유지

### 6. 시나리오 시작과 AI 첫 발화

- **Given** — 미리보기에서 "시작하기"를 눌렀고 새 진행 중 인스턴스가 생성됐다
- **When** — 채팅 화면이 로드된다
- **Then** — AI가 시나리오 캐릭터로 첫 메시지를 보낸 상태로 화면이 나타나고, 채팅 헤더 아래에 3개 목표가 모두 ⚪ 상태로 표시되며, 입력창에 포커스가 가 있다

성공 기준:
- [ ] 진입 시 AI의 첫 메시지 버블이 이미 표시되어 있다 (사용자가 먼저 입력하지 않아도)
- [ ] AI 메시지에 캐릭터 이름 라벨(예: "Barista")이 표시되지 않는다
- [ ] 채팅 헤더 아래에 목표 3개가 체크박스(⚪) 형태로 표시된다
- [ ] 입력창의 placeholder는 "영어로 메시지 입력... (막히면 한국어로 써도 돼요)"
- [ ] 입력창에 자동으로 키보드 포커스가 들어 있다

### 7. 영어 메시지 — ✓ (정상)

- **Given** — 진행 중 채팅에서 입력창에 "Hi, can I order a latte?"가 입력된 상태
- **When** — Enter 키를 누른다
- **Then** — 사용자 메시지 버블이 표시되고, AI 응답이 짧은 1-2문장으로 스트리밍되며, 사용자 메시지에는 별도의 라벨드 섹션 없이 ✓ 마크만 붙는다

성공 기준:
- [ ] 사용자 메시지 텍스트는 입력한 그대로 ("Hi, can I order a latte?") 표시된다
- [ ] AI 응답이 토큰 단위로 점진적 표시(스트리밍)된다
- [ ] AI 응답에 "[smiles]", "*nods*" 같은 메타/내레이션 텍스트가 포함되지 않는다
- [ ] 분석 완료 후 사용자 버블에 ✓가 붙으며, 별도의 교정 라벨드 섹션은 표시되지 않는다

### 8. 영어 메시지 — ✱ (교정 필요)

- **Given** — 진행 중 채팅
- **When** — 사용자가 "I go to school yesterday"를 입력하고 전송한다
- **Then** — 사용자 메시지 버블이 표시되고, AI는 캐릭터 안에서 자연스럽게 응답을 스트리밍하며, 잠시 후 사용자 버블에 ✱ 라벨드 섹션(기본 펼침)이 추가되어 교정문 "I went to school yesterday"와 한국어 설명이 표시된다

성공 기준:
- [ ] 사용자 원본 텍스트는 변형 없이 "I go to school yesterday"로 유지된다
- [ ] AI 응답이 ✱ 마크 도착보다 먼저 시작될 수 있다 (병렬 스트리밍 — 분석을 기다리며 채팅이 멈추지 않는다)
- [ ] 같은 버블 안에 "✱ 교정" 헤더가 있는 라벨드 섹션이 펼친 상태로 나타나고, 교정문과 한국어 설명이 보인다
- [ ] 섹션 헤더 우측에 [⌃/⌄] 토글이 있고, 토글하면 섹션 본문이 접히거나 펼쳐진다
- [ ] 한 버블에 ✱와 💡가 동시에 표시되지 않는다 (오류가 무거우면 ✱)
- [ ] AI 응답은 사용자가 보낸 **원본** 텍스트에 대해 답한다 (예: "I go yesterday"에 시제 보정 없이 캐릭터로 응답)

### 9. 영어 메시지 — 💡 (대안)

- **Given** — 진행 중 채팅
- **When** — 사용자가 의미는 통하지만 textbook 같은 "I would like to purchase coffee, please"를 전송한다
- **Then** — 사용자 버블에 💡 라벨드 섹션이 추가되어 "Can I get a coffee?" 같은 더 자연스러운 대안과 한국어 설명이 표시된다

성공 기준:
- [ ] 💡 섹션 헤더 텍스트 ("💡 대안" 등)가 ✱와 구분된다
- [ ] 본문에 대안 문장 + 한국어 부연 설명이 보인다
- [ ] AI 응답은 원본 텍스트 ("I would like to purchase coffee, please")에 대해 답한다

### 10. 마이너 오타는 통과

- **Given** — 진행 중 채팅
- **When** — 사용자가 "I want a coffe"(오타) 같은 가벼운 오타가 있는 문장을 전송한다
- **Then** — 사용자 메시지에는 ✓ 마크가 붙고, 별도 교정 섹션이 표시되지 않는다

성공 기준:
- [ ] 단순 오타는 ✱를 발동시키지 않는다 (학습 가치 없음, 원어민 이해 가능)
- [ ] 사용자 버블에 ✓만 표시되고 라벨드 섹션은 없다

### 11. 한국어 입력 → 영어 변환 (전송 전 미리보기)

- **Given** — 진행 중 채팅, 입력창이 비어 있는 상태
- **When** — 입력창에 "학교 갔어"를 입력하고 잠시 멈춘다
- **Then** — 입력창 위에 카드 형태로 영어 미리보기 ("→ I went to school")가 나타나고, 옆에 "이걸로 보내기" / ↗ 버튼이 있다

성공 기준:
- [ ] 입력에 한글 문자가 포함되면 짧은 디바운스 후 변환 미리보기 카드가 입력창 위에 나타난다
- [ ] 미리보기 카드의 "보내기" 버튼 클릭 → 메시지가 전송되고, 채팅에 버블이 추가된다
- [ ] 전송된 버블의 메인 텍스트는 한글 원본 ("학교 갔어")이고, 그 아래 🌐 라벨드 섹션(펼침)에 영어 번역 ("I went to school")이 보인다
- [ ] 🌐 섹션이 표시된 버블에 ✱ 또는 💡 마크는 붙지 않는다 (본인이 쓴 게 아니므로 교정 대상 아님)
- [ ] AI는 영어로 변환된 문장에 대해 응답한다 (외부 관찰: 응답이 "I went to school" 의미에 맞춰진다)

### 12. 한국어 미리보기가 떠 있을 때 Enter 동작

- **Given** — 한글이 포함된 입력으로 미리보기 카드가 나와 있는 상태
- **When** — 사용자가 Enter(전송)를 누른다
- **Then** — 미리보기의 영어 번역이 전송되고, 한글만 단독으로 채팅에 들어가지 않는다

성공 기준:
- [ ] 미리보기가 떠 있는 상태에서 Enter → 영어 변환문이 전송된다
- [ ] 채팅에 한글만 들어간 (영어 섹션 없는) 사용자 버블은 생성되지 않는다

### 13. 목표 달성 평가 (실시간)

- **Given** — 카페 시나리오 진행 중. 목표는 "1. 음료 주문하기", "2. 사이즈 명시하기", "3. 결제 의사 표현하기". 셋 다 ⚪ 상태
- **When** — 사용자가 "Can I get a small latte, please?"를 전송한다
- **Then** — 채팅 헤더 아래 목표 카드 중 "1. 음료 주문하기"와 "2. 사이즈 명시하기"가 ✓로 바뀌며 짧은 펄스/색 변화가 발생한다

성공 기준:
- [ ] 메시지 전송 후 분석이 완료되면 해당 메시지로 달성된 목표가 ⚪ → ✓로 전환된다
- [ ] 한 메시지로 여러 목표가 동시에 달성될 수 있다
- [ ] 달성된 목표가 다음 메시지에서도 ✓ 상태로 유지된다 (역행하지 않음)
- [ ] 달성 전환 시 짧은 시각 효과(펄스/색 변화)가 일어난다

### 14. 모든 목표 달성 → 종료 권유

- **Given** — 진행 중 채팅에서 마지막 목표가 막 ✓로 전환됐다
- **When** — 분석이 완료되고 잠시 후
- **Then** — "목표 다 했어요! 끝낼까요?" 권유 모달이 나타나고, "끝내기" / "더 대화" 두 버튼이 보인다

성공 기준:
- [ ] 3/3 달성 직후 권유 모달이 한 번 자동 표시된다
- [ ] "끝내기" → 시나리오 인스턴스가 "종료됨" 상태로 전환, 사이드바 진행 중에서 사라지고 히스토리에 나타난다
- [ ] "더 대화" → 모달이 닫히고 채팅이 계속된다. 목표는 3/3 ✓ 상태로 헤더에 그대로 유지
- [ ] 자동으로 종료되거나 입력창이 막히지 않는다 (사용자 통제권 유지)

### 15. 더 알아보기 — 사이드 대화 열기

- **Given** — 사용자 메시지에 ✱ 섹션이 펼쳐져 있다
- **When** — 펼쳐진 섹션 안의 "더 알아보기 →" 버튼을 누른다
- **Then** — 웹에서는 화면 우측에서 폭 ~420px의 드로어가 슬라이드인, 모바일에서는 화면 하단에서 50-70% 높이의 바텀시트가 올라오며, 상단에 "맥락 중인 교정" 카드와 시드 메시지 "이 교정에 대해 궁금한 점이 있나요?" + 자주 묻는 질문 칩 3개가 보인다

성공 기준:
- [ ] 데스크탑 폭에서는 우측 드로어, 모바일 폭에서는 바텀시트 형태로 열린다
- [ ] 헤더에 어떤 ✱에 대해 묻는지 보여주는 카드가 표시된다
- [ ] 시드 영역에 "이 교정에 대해 궁금한 점이 있나요?" 와 칩 3개 ("왜 이게 틀려?", "다른 표현은?", "비슷한 예시")가 보인다
- [ ] 사이드 패널 영역 바깥(메인 채팅)을 클릭해도 패널이 닫히지 않는다
- [ ] X 버튼 또는 Esc 키 → 패널이 닫힌다

### 16. 더 알아보기 — 코치 페르소나 응답

- **Given** — 더 알아보기 패널이 열린 상태
- **When** — 칩 "왜 이게 틀려?"를 누르거나 직접 한국어로 질문을 입력해 전송한다
- **Then** — AI 코치가 한국어로 답변을 스트리밍하며, 영어 예시는 영어 그대로 인용한다

성공 기준:
- [ ] 코치 응답의 설명문은 한국어로 표시된다
- [ ] 영어 예시 인용은 영어 형태로 (번역하지 않은 채) 표시된다
- [ ] 패널 안의 대화는 메인 채팅에 새 버블을 추가하지 않는다 (격리)
- [ ] 메인 채팅의 목표 진행 상태가 바뀌지 않는다

### 17. 더 알아보기 — 닫으면 소실

- **Given** — 더 알아보기 패널에서 몇 차례 질문·답변을 주고받았다
- **When** — X 또는 Esc로 패널을 닫은 뒤, 같은 ✱ 섹션의 "더 알아보기"를 다시 연다
- **Then** — 패널이 다시 시드 메시지 상태에서 시작되고, 이전 대화는 보이지 않는다

성공 기준:
- [ ] 같은 ✱의 사이드 대화를 다시 열어도 이전 메시지 이력이 표시되지 않는다
- [ ] 다시 시드 메시지와 칩 3개가 보인다

### 18. 한 시나리오 인스턴스가 진행 중일 때 같은 시나리오 다시 열기

- **Given** — "카페 주문" 시나리오에 진행 중 인스턴스가 1개 있다 (사이드바에 ▸ 표시)
- **When** — 사이드바에서 "카페 주문" 시나리오를 클릭한다
- **Then** — 메인 영역에 "이어서 하기" / "새로 시작" 두 옵션이 표시된다

성공 기준:
- [ ] 진행 중 인스턴스가 있는 시나리오 클릭 → 두 옵션 화면
- [ ] "이어서 하기" → 기존 인스턴스의 채팅으로 진입 (이전 메시지·목표 상태 그대로)
- [ ] "새로 시작" → 기존 진행 중 인스턴스 처리 방식은 미결정 (현 가정: 기존 인스턴스가 자동 "종료됨"으로 전환되고 새 인스턴스 시작)
- [ ] 시나리오당 동시에 진행 중 인스턴스는 최대 1개

### 19. 중도 종료

- **Given** — 진행 중 채팅. 목표 1/3 달성 상태
- **When** — 사용자가 채팅 헤더의 "종료" 메뉴를 누른다
- **Then** — "목표 1/3 달성, 정말 종료할까요?" 확인 모달이 나타난다

성공 기준:
- [ ] 확인 모달에 현재 달성률("1/3")이 표시된다
- [ ] "취소" → 모달 닫힘, 채팅 계속
- [ ] "종료" → 시나리오 인스턴스가 "진행 중" 상태로 유지 (히스토리로 가지 않음). 사이드바에서 ▸ 표시는 유지된다

### 20. 히스토리 항목 클릭 — 읽기 전용 보기

- **Given** — 사이드바 "히스토리" 섹션에 종료된 인스턴스가 있다
- **When** — 그 항목을 클릭한다
- **Then** — 메인 영역에 과거 대화 전체(원본 메시지 + 교정 라벨드 섹션 + 한→영 번역 섹션 + AI 응답)가 그대로 렌더되고, 입력창은 비활성화 상태이며 상단에 "다시 시작하기" 버튼이 있다

성공 기준:
- [ ] 채팅 입력창이 비활성화 상태로 보이거나 표시되지 않는다
- [ ] 메시지 버블의 ✱/💡/🌐 섹션은 펼침/접힘 토글 가능
- [ ] ✱ 섹션 안의 "더 알아보기" 버튼은 클릭 가능 (별도 코치 대화 — 메인 대화에 영향 없음)
- [ ] 상단에 "다시 시작하기" 버튼이 있고, 클릭 시 같은 시나리오의 새 인스턴스가 시작된다
- [ ] 같은 시나리오에 진행 중 인스턴스가 이미 있으면, "다시 시작하기" 클릭 시 시나리오 18 흐름과 동일하게 옵션 화면으로 안내

### 21. 시나리오 삭제

- **Given** — 사이드바에 시나리오가 있다 (진행 중이거나 종료됨)
- **When** — 사이드바 항목 위에서 호버 메뉴를 열고 "삭제"를 누른다
- **Then** — "이 시나리오와 모든 대화 기록을 삭제할까요?" 확인 모달이 나타나고, 확인 시 시나리오와 관련된 모든 인스턴스·메시지가 사이드바에서 사라진다

성공 기준:
- [ ] 진행 중·종료된 시나리오 모두 삭제 메뉴가 보인다
- [ ] 확인 후 시나리오가 사이드바 "내 시나리오" 섹션에서 사라진다
- [ ] 해당 시나리오의 모든 히스토리 항목도 함께 사라진다
- [ ] 삭제는 되돌릴 수 없다 (확인 모달이 유일한 보호 장치)

### 22. 시나리오 편집

- **Given** — 시나리오 카드(진행 중 또는 종료) 호버 메뉴
- **When** — "편집"을 선택한다
- **Then** — 생성과 동일한 폼이 메인 영역에 인라인으로 열리고, 기존 값이 채워져 있다

성공 기준:
- [ ] 편집 폼은 생성 폼과 동일한 필드 구성·검증을 따른다
- [ ] 폼 저장 → 시나리오 정의가 갱신되고 사이드바에 반영된다
- [ ] 편집 후에도 기존 인스턴스(진행 중·종료)의 대화 내용은 보존된다

### 23. 채팅 입력 동작

- **Given** — 진행 중 채팅, 입력창에 포커스가 있다
- **When** — 사용자가 "Hello"를 입력하고 Shift+Enter를 누른 후 "How are"를 입력한다
- **Then** — 입력창이 줄바꿈을 포함해 자동으로 확장된다

성공 기준:
- [ ] Enter 단독 → 전송
- [ ] Shift+Enter → 줄바꿈
- [ ] 한글 IME 조합 중 Enter → 전송되지 않는다 (조합 완료 후 다시 Enter해야 전송)
- [ ] 입력창 높이는 1줄에서 최대 5줄까지 자동 확장
- [ ] 입력창이 비어 있으면 종이비행기 전송 버튼은 비활성
- [ ] 입력 길이가 500자에 다다르면 시각적 경고 표시 (입력은 가능)
- [ ] AI 응답이 스트리밍 중이면 전송 버튼이 비활성화되고 Enter 키로도 전송되지 않는다

### 24. 채팅 호출 실패 — 다시 시도

- **Given** — 진행 중 채팅
- **When** — 사용자가 메시지를 전송했으나 채팅 응답 호출이 실패한다 (네트워크/서버 오류 또는 30초 타임아웃)
- **Then** — 사용자 메시지 버블 옆에 "다시 시도" 버튼이 표시되고, 사용자가 입력한 텍스트는 보존된다

성공 기준:
- [ ] 사용자 버블은 사라지지 않는다
- [ ] "다시 시도" 클릭 → 같은 텍스트로 다시 전송 시도
- [ ] 실패가 반복되어도 채팅 상태가 깨지지 않는다 (앱이 무한 로딩에 빠지지 않음)

### 25. 분석 호출 실패 — 조용한 fallback

- **Given** — 진행 중 채팅. 사용자가 메시지를 보냈고 채팅 응답은 정상 도착했다
- **When** — 분석 호출이 실패한다 (또는 10초 타임아웃)
- **Then** — 사용자 버블에는 마크가 붙지 않은 채로 진행되고, 사용자에게 별도의 에러 표시가 보이지 않으며, 채팅은 계속된다

성공 기준:
- [ ] 분석 실패 시 사용자 버블에 ✱/💡/✓ 어떤 마크도 표시되지 않는다 (조용히 통과)
- [ ] 목표 진행 상태는 갱신되지 않는다 (이전 상태 유지)
- [ ] 다음 메시지를 정상적으로 보낼 수 있다
- [ ] 화면 어딘가에 "분석 사용 불가" 같은 큰 에러 메시지가 보이지 않는다

### 26. 사이드바 토글 (데스크탑)

- **Given** — 데스크탑 화면 폭, 사이드바가 펼쳐져 있다 (260px)
- **When** — 사이드바 상단의 접기 버튼을 누른다
- **Then** — 사이드바가 접히고 메인 영역이 전체 폭으로 확장된다

성공 기준:
- [ ] 접힌 상태에서도 사이드바를 다시 펼 수 있는 트리거가 보인다
- [ ] 접힘/펼침 상태는 사용자가 명시적으로 바꿀 때까지 유지된다 (세션 내)

### 27. 사이드바 (모바일)

- **Given** — 모바일 화면 폭
- **When** — 상단 햄버거 메뉴 아이콘을 누른다
- **Then** — 사이드바가 오버레이로 슬라이드인되며, 시나리오 항목을 누르면 사이드바가 닫히고 메인이 해당 화면으로 전환된다

성공 기준:
- [ ] 모바일 폭에서는 하단 탭바 형태가 아닌 사이드 오버레이로 표시된다
- [ ] 시나리오 클릭 → 사이드바 자동 닫힘 + 메인 전환
- [ ] 오버레이 바깥 영역 탭 → 사이드바 닫힘

### 28. 히스토리 길어지면 전체 보기

- **Given** — 사이드바 히스토리 항목이 30개를 넘긴다
- **When** — 사이드바 히스토리 섹션을 본다
- **Then** — 최근 항목만 보이고 하단에 "전체 보기" 링크가 있다

성공 기준:
- [ ] 사이드바에는 항목 수가 일정(30개) 이하로 유지된다
- [ ] "전체 보기" 클릭 → 별도 페이지/뷰로 이동, 전체 히스토리 목록이 표시된다

## 불변 규칙

### 보안 / 프라이버시
- 로그인한 본인 user_id에 귀속된 데이터(시나리오·인스턴스·메시지)만 어떤 경로(사이드바·URL 직접 접근·검색)로도 접근 가능하다. 다른 user_id의 데이터는 보이지 않는다.

### 데이터 일관성
- 사용자가 보낸 메시지의 메인 텍스트는 입력 그대로 보존된다 (한국어 입력의 경우 한국어 원본). 교정문은 별도의 라벨드 섹션에만 표시되며 원본을 덮어쓰지 않는다.
- 메시지당 마크는 정확히 하나다 (✓, ✱, 💡, 🌐 중 하나). 마이너 오타는 ✓에 흡수, ✱와 💡가 동시에 나오지 않는다 (✱ 우선).
- 한 번 ✓로 전환된 목표는 같은 인스턴스 내에서 ⚪로 되돌아가지 않는다.

### 응답 속도 / 동시성
- 사용자 메시지 전송 후, 채팅 응답의 첫 토큰이 도착하기 전에 분석 결과(✱/💡)를 기다리지 않는다 — 채팅이 분석에 블록되지 않는다 (병렬 진행).
- AI 응답이 스트리밍 중일 때는 다음 메시지가 전송되지 않는다 (큐잉 없음).

### 격리
- 더 알아보기 사이드 패널 내부의 대화는 메인 채팅의 메시지 흐름·목표 상태에 어떤 영향도 주지 않는다.
- 더 알아보기 패널을 닫으면 내부 대화는 즉시 폐기된다.
